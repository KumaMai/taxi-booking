Comparison of deployment options A and B for Next.js taxi booking system

Option A

Vercel + Neon PostgreSQL

Zero-infrastructure, fully managed cloud

1
สร้าง repo บน GitHub
push code ขึ้น GitHub ปกติbranch: main = production, dev = staging
2
สมัคร Neon (neon.tech)
สร้าง PostgreSQL database ออนไลน์ได้ connection string เช่น postgresql://user:pass@ep-xxx.neon.tech/taxi
3
สมัคร Vercel (vercel.com)
กด "Import GitHub Repository"Vercel อ่าน Next.js project ได้อัตโนมัติ ไม่ต้อง config
4
ตั้ง Environment Variables
ใส่ใน Vercel DashboardDATABASE_URL, NEXTAUTH_SECRET, EMAIL_PASS
5
Run Prisma Migration
npx prisma db push ครั้งแรกPrisma สร้าง table ให้อัตโนมัติจาก schema.prisma
6
ผูก Custom Domain
ชี้ DNS ที่ domain registrar → VercelSSL cert ออกอัตโนมัติ ไม่ต้องทำอะไรเพิ่ม
7
Deploy อัตโนมัติทุก push
git push origin main → live ใน ~2 นาทีPreview URL ทุก branch/PR โดยอัตโนมัติ
ข้อดี
  • ไม่ต้องดูแล server เลย
  • SSL อัตโนมัติ
  • Global CDN ทั่วโลก
  • Auto-scale ตาม traffic
  • Preview deploy ทุก branch
  • Neon backup อัตโนมัติ
  • Uptime 99.99% SLA
ข้อเสีย
  • Vendor lock-in (Vercel)
  • Neon cold start ~500ms
  • ค่าใช้จ่ายเพิ่มตาม traffic
  • API route timeout 10-30s
  • ไม่ control server ได้
  • Background jobs ทำยาก

💰 ค่าใช้จ่าย: $0/เดือน (Hobby) → $20/เดือน (Pro) + Neon Free tier

Option B

Docker Compose + VPS (Production)

Self-hosted, full control, GitHub Actions CI/CD

1
เช่า VPS (DigitalOcean / Vultr / Hetzner)
Ubuntu 22.04 LTS, 2GB RAM แนะนำ$6-12/เดือน — ติดตั้ง Docker + Docker Compose
2
เขียน Dockerfile + docker-compose.prod.yml
Next.js app + PostgreSQL + NginxNginx ทำหน้า reverse proxy + SSL termination
3
ตั้ง SSL ด้วย Let's Encrypt + Certbot
certbot --nginx -d yourdomain.comฟรี ต่ออายุอัตโนมัติทุก 90 วัน
4
ตั้ง GitHub Actions CI/CD
push → build Docker image → SSH deploy.github/workflows/deploy.yml จัดการทั้งหมด
5
ตั้ง Backup อัตโนมัติ
cron job dump PostgreSQL ทุกคืนpg_dump → upload S3 / Backblaze B2
6
ตั้ง Monitoring
UptimeRobot (ฟรี) แจ้งเตือนถ้า site down+ Docker restart policy: unless-stopped
7
Deploy ครั้งแรก
docker compose up -d บน VPSครั้งถัดไป push code → GitHub Actions deploy อัตโนมัติ
ข้อดี
  • ควบคุมได้ 100%
  • ไม่มี vendor lock-in
  • ราคาคงที่ ไม่เพิ่มตาม traffic
  • ไม่มี cold start
  • Background jobs ได้ปกติ
  • ข้อมูลอยู่บน server ตัวเอง
  • Upgrade ทำเมื่อไรก็ได้
ข้อเสีย
  • ต้อง setup เองทั้งหมด
  • ต้อง patch OS เอง
  • Backup ต้องตั้งเอง
  • Scale ได้ยากถ้า traffic พุ่ง
  • ถ้า VPS down = site down
  • ต้องใช้เวลา setup ~4-6h

💰 ค่าใช้จ่าย: $6/เดือน (1GB) → $12/เดือน (2GB RAM) คงที่ตลอด
